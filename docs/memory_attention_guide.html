SAM2 Memory Attention 微调：困难与解决

记录在声呐 B-scan 数据上微调 Memory Attention 遇到的技术问题和解决方案

为什么需要微调 Memory Attention？

SAM2 在做视频跟踪时，流程如下：

帧0: image → encoder → prompt → mask_decoder → mask ↓ memory_encoder → 存入 Memory Bank ↓ 帧1: image → encoder → MemoryAttention(靠 Memory Bank 条件化) → mask_decoder → mask ↓ memory_encoder → 更新 Memory Bank ↓ 帧2: image → encoder → MemoryAttention(靠更新后的 Memory Bank) → ...

核心矛盾：Memory Attention 在自然视频上训练（平滑运动），但声呐目标是低速、离散、忽隐忽现的小亮点。冻结的 Memory Attention 用"平滑运动"的假设去匹配声呐的"跳跃运动"→ 特征匹配失败 → 跟踪漂移。

困难时间线

困难 1：MemoryAttention.forward() 接口不兼容

第一版直接调 self.model.memory_attention(tgt=..., memory=..., pos=..., query_pos=...)，报错 got unexpected keyword argument 'tgt'

根因：MemoryAttention 的参数名是 (curr, memory, curr_pos, memory_pos)，不是 (tgt, pos, query_pos)。文档和源码不一致。
困难 2：cross_attn 维度不匹配

修正参数名后，报错 mat1 and mat2 shapes cannot be multiplied (4096x256 and 64x256)

MemoryAttention 内部有两套注意力： ┌─────────────────────────────────────────────────┐ │ self_attn: q_proj(256→256) k_proj(256→256) │ ← 兼容 backbone 特征 (256维) │ v_proj(256→256) out_proj(256→256) │ ├─────────────────────────────────────────────────┤ │ cross_attn: q_proj(256→256) k_proj(64→256) │ ← 期望 memory_encoder 输出 (64维) │ v_proj(64→256) out_proj(256→256) │ 不能直接喂 backbone 特征 └─────────────────────────────────────────────────┘
Our attempt to pass backbone features (256-dim) to cross_attn's k_proj (expects 64-dim) failed with dimension mismatch.
困难 3：memory_encoder 输出格式不可达

要获得 64 维的 memory 特征，必须通过 memory_encoder：

memory_encoder(
    pix_feat=vis_feats,             // backbone 特征
    maskmem_features=...,           // 需要 SAM2VideoPredictor 内部的 mask memory
    maskmem_pos_enc=...,            // 需要 SAM2VideoPredictor 内部的 memory 位置编码
)
这些输入只在 SAM2VideoPredictor 的 @torch.inference_mode() 上下文中可用。训练模式下无法访问。
解决：只训练 self_attn

绕开 cross_attn 和 memory_encoder，直接调用 Memory Attention 每一层的 _forward_sa() 方法：

for layer in self.model.memory_attention.layers:
    feat = layer._forward_sa(feat, query_pos)

这样 self_attn 的 q_proj/k_proj/v_proj/out_proj（全部 256 维）被正常训练，gradient 流动无误。推理时，cross_attn 仍然使用预训练权重，self_attn 已经适配了声呐特征。

困难 4：clip 训练时 DataLoader 维度混乱

clip 模式下 Dataset 返回 (T, 3, H, W)，DataLoader 又套了一层 batch 维度变成 (1, T, 3, H, W)。不加 squeeze 时在 forward_clip 里维度对不上。

解决：训练循环中检测 clip_mode，做 images.squeeze(0)。
困难 5：clip 模式不支持标准数据增强

单帧模式下水平翻转、亮度抖动等增强按帧随机应用。clip 模式下 4 帧是连续的，不能随机翻转/打乱——会破坏帧间一致性。

解决：clip 模式下跳过数据增强（仅做归一化），保持帧间物理一致性。
