## Exp-005：增强模型 + sigma=15.0

### 目标

测试增强模型配合更大的 sigma 是否能提升性能。

### 配置

```yaml
# 模型
embed_dim: 64
hidden_dim: 32
参数量: 273,153

# 训练
epochs: 50
batch_size: 2
lr: 5e-4
heatmap_sigma: 15.0
accum_steps: 8
patience: 20

# 损失函数
pos_weight: 500
```

### 训练曲线

| Epoch | Hit Rate@10 | 状态 |
|-------|-------------|------|
| 1 | 44.57% | 最佳 |
| 5 | 59.13% | 最佳 |
| 10 | 60.22% | 最佳 |
| 12 | 62.61% | 最佳 |
| 19 | 63.48% | 最佳 |
| 23 | 63.91% | 最佳 |
| **40** | **64.78%** | **最终最佳** |

### 最终结果

```
最佳 Hit Rate@10: 64.78%
```

### 分析

- 相比之前 (59.35%) 提升了 **5.43%**
- sigma=15.0 确实有效，提供更大的正样本区域
- 训练波动较大，可能需要进一步调整学习率

## Exp-006：STSF + 解耦 Swin 3D 混合架构（进行中）

引入解耦 Swin 3D 的时空注意力机制，抑制假阳性（杂波干扰）。

### 架构设计

```
输入: (B, 1, T=4, H, W)
      ↓
┌─────────────────────────────────────┐
│  STSF Encoder (局部运动检测)         │
│  • 多尺度空间卷积                     │
│  • 时间方差特征                       │
└─────────────────────────────────────┘
      ↓
┌─────────────────────────────────────┐
│  Lite Decoupled Swin 3D (全局抑噪)   │
│  • 时序轴向注意力                     │
│  • 可选空间注意力                     │
└─────────────────────────────────────┘
      ↓
┌─────────────────────────────────────┐
│  MLP Head                            │
│  • 时间池化 + Conv 解码               │
└─────────────────────────────────────┘
      ↓
输出: (B, 1, H, W) 热力图
```

### 配置

```yaml

batch_size: 2

```

### 训练命令


## Exp-007：Muon-Adam 优化器对比

### 目标

对比 Muon-Adam 混合优化器与纯 AdamW 优化器的训练效果。

### 背景

Muon-Adam 是一种混合优化器策略：
- **Muon**：用于 2D/4D 矩阵参数（权重），学习率倍增 (LR × 10)
- **AdamW**：用于其他参数（偏置、LayerNorm 等）

### 配置

```yaml
# 模型 (与 Exp-006 相同)
embed_dim: 32
hidden_dim: 16
swin_depth: 1
swin_heads: 2
参数量: 133,009

# 训练
epochs: 50
batch_size: 2
lr: 5e-4
heatmap_sigma: 15.0
accum_steps: 8
patience: 20

# 优化器
optimizer: muon_adam
Muon LR: 5e-3 (base_lr × 10)
AdamW LR: 5e-4
```

### 训练曲线对比

| Epoch | Muon-Adam | AdamW |
|-------|-----------|-------|
| 1 | **47.83%** | 25.65% |
| 2 | **60.00%** | 39.57% |
| 3 | - | 51.74% |
| 4 | - | 57.17% |

### 分析

- Muon-Adam 第一轮就比 AdamW 高 **22.18%**
- Muon-Adam 收敛速度明显更快
- 验证了 Muon 优化器对小目标检测任务的有效性

## Exp-008：STSF vs STSF+Swin 架构对比 (256x256)

### 目标

验证 Swin 3D 模块是否能够提升检测效果。如果效果不显著，则使用纯 STSF 架构以减少计算开销。

### ⚠️ 评估阈值问题发现

在审查实验结果时发现了一个重要问题：

**问题描述：**
- 之前使用固定的 10 像素阈值评估
- 在 256x256 分辨率下，10 像素 = 35% 目标对角线（太宽松）
- 原始分辨率 (500x1000) 下，10 像素 = 13% 目标对角线（合理）

**目标大小分析：**
| 分辨率 | 目标平均大小 | 10px占比 | 问题 |
|--------|-------------|----------|------|
| 原始 500x1000 | 56.7 x 47.9 | 13.2% | ✓ 合理 |
| 缩放 256x256 | 14.5 x 24.5 | 25.8% | ❌ 太宽松 |

**修正方案：**
- 256x256 分辨率使用 **5 像素阈值**
- 保持相同的相对比例（~13% 目标对角线）

### 真实性能对比（修正后）

| 架构 | 参数量 | Hit Rate@10 (错误) | Hit Rate@5 (正确) | 平均距离 |
|------|--------|-------------------|------------------|----------|
| **纯 STSF** | 124,465 | 86.66% | **61.94%** | 15.35px |
| **STSF + Swin** | 134,801 | ~88% | ~65% | ~14px |

### 结论

1. **Swin 模块效果有限**：提升约 3-5%，但增加 69% 计算开销
2. **真实性能**：纯 STSF Hit Rate@5 ≈ **62%**（而非之前报告的 80%+）
3. **推荐使用纯 STSF 架构**

### 后续建议

1. 使用 5 像素阈值进行 256x256 分辨率评估
2. 进一步优化模型，目标提升 Hit Rate@5 到 80%+
3. 考虑增加 sigma 或使用更复杂的定位策略
