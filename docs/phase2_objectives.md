## 为什么选择 80% 作为目标？

### 1. SAM 2 容错能力分析

SAM 2 的 Prompt 点容错能力：
- **< 10px 偏差**：几乎 100% 能正确分割目标
- **10-20px 偏差**：大部分情况能正确分割，但边缘可能不准确
- **> 20px 偏差**：可能分割到背景或错误区域

### 2. 失败率控制

- 80% 成功率 = 20% 失败率
- 失败的 20% 可通过 **Re-prompting 机制** 补救
- Re-prompting 在阶段三实现

### 3. 实际应用场景

声呐小目标检测场景特点：
- 目标小（通常 < 50px）
- 视频帧率低（通常 10-30 FPS）
- 允许少量人工干预或延迟

### 4. 与阶段三的关系

```
阶段二输出 → SAM 2 的 Prompt 点

┌─────────────────────────────────────────────────────┐
│  预测点位置         │  SAM 2 行为                   │
├─────────────────────────────────────────────────────┤
│  成功（< 10px）     │  正确初始化跟踪，后续帧正常    │
│  失败（> 10px）     │  跟踪错误区域，需要 Re-prompt  │
└─────────────────────────────────────────────────────┘
```
