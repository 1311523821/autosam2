SAM 2 LoRA 微调完全指南

从零理解如何用 LoRA 微调 SAM 2 处理声呐 B-scan 小目标检测

1. 为什么要微调 SAM 2？

SAM 2 在自然图像上预训练。它见过猫、狗、汽车，但从未见过声呐 B-scan 图像。

核心矛盾：冻结的 Image Encoder 不认识声呐图像中的目标，它输出给 Decoder 的是"垃圾特征"。Decoder 再强也无法从垃圾特征中画出正确的 Mask。

3. 什么是 LoRA

3.1 传统微调 vs LoRA 微调

传统全参数微调

直接修改模型所有参数。

❌ 参数量巨大（SAM 2 Tiny 约 39M）

❌ 显存开销大

❌ 容易过拟合（数据少时）

❌ 需要保存完整模型副本

LoRA 微调

冻结原始权重，只训练低秩旁路矩阵。

✅ 可训练参数仅 ~0.1M（0.3%）

✅ 显存几乎不增加

✅ 防过拟合（参数少 + 低秩约束）

✅ 权重可直接合并到原模型

3.2 LoRA 的数学原理

原始线性层的计算：

输入 x (d维) ──→ [ W (d×d) ] ──→ 输出 h (d维) ↑ 39M 参数，全部冻结

LoRA 在旁边加一条低秩路径：

┌─ 原始路径（冻结） ─┐ 输入 x ──→ ─────┤ W·x (d×d矩阵乘) ├──────→ 输出 h + Δh │ │ └─ LoRA 路径（可训练） ─┘ A (r×d) → B (d×r) ↑ ↑ 秩 r=4 秩 r=4 参数: 4d 参数: 4d 总共: 8d (vs 原始 d²) 公式：output = W·x + (x @ A^T @ B^T) × α/r ↑ ↑ 原始输出 LoRA 修正量

为什么秩 r=4 就够了？

W 的改变量 ΔW 虽然维度大（d×d），但 ΔW 的本质信息量很低——它只需要描述"声呐图像 vs 自然图像"的差异。这种差异可以用低秩矩阵近似。r=4 意味着只用 4 个独立方向就足够表达这种差异。

            loss.backward()                           # 6. 反向传播

            optimizer.step()                          # 8. 更新参数

    # ── 定期验证 ──
    if epoch % val_every == 0:
        val_loss = validate(test_videos)              # 单帧loss（非端到端）
        if val_loss < best_val_loss:
            save_checkpoint(best_train)               # 含LoRA结构，可恢复训练

为什么用单帧训练而不是视频训练？
视频训练需要在多帧间传递梯度（Memory Attention），极其消耗显存且不稳定。单帧训练把所有帧视为独立样本，训练稳定高效。微调完成的权重直接用于 Video Predictor 推理，跟踪效果自然变好——因为 "识别" 部分变强了，"跟踪" 部分不需要训练。

6. 关键代码解析

6.1 LoRA 层的实现

class LoRALinear(nn.Module):
    """在原始 Linear 层旁边加低秩旁路"""
    def __init__(self, original_linear, r=4, lora_alpha=8):
        # 保存原始层（冻结）
        self.original_linear = original_linear
        self.original_linear.weight.requires_grad = False

        # 创建低秩矩阵 A (r×d) 和 B (d×r)
        in_dim  = original_linear.in_features   # 例如 96
        out_dim = original_linear.out_features  # 例如 288
        self.scaling = lora_alpha / r            # 8 / 4 = 2.0

        self.lora_A = nn.Parameter(torch.zeros(r, in_dim))   # (4, 96)
        self.lora_B = nn.Parameter(torch.zeros(out_dim, r))   # (288, 4)

        # 初始化：A 用 Kaiming，B 初始化为零
        nn.init.kaiming_uniform_(self.lora_A)
        nn.init.zeros_(self.lora_B)

    def forward(self, x):
        # 原始输出（冻结，无梯度）
        original = self.original_linear(x)         # x @ W^T

        # LoRA 修正量（可训练，有梯度）
        lora = (x @ self.lora_A.T @ self.lora_B.T) * self.scaling

        # 最终输出 = 原始 + LoRA修正
        return original + lora

6.2 为什么 B 初始化要为零？

训练开始时： B = 0 → lora_out = x @ A^T @ 0 = 0 → output = original + 0 第一轮反向传播： B 的梯度 ≠ 0（因为 loss 对 output 有梯度） A 的梯度 = 0（因为 B × A 中 B=0，d(loss)/dA = 0） ← 这是正常的！ 第二轮开始： B 已被更新为非零值 → A 的梯度 ≠ 0 → 两者都开始正常训练 ✅ 这样设计的好处：LoRA 从"不做任何修改"开始， 逐步学习需要的修正量，不会破坏预训练权重。

6.3 保存与推理

def save_for_inference(save_path):
    """保存可直接用于 Video Predictor 推理的权重"""
    clean_state_dict = {}

    for key, value in model.state_dict().items():
        if 'lora_A' in key or 'lora_B' in key:
            continue  # 跳过 LoRA 参数，推理时不需要

        if 'original_linear.weight' in key:
            # 合并 LoRA 权重：W_merged = W + B×A × scale
            # 然后把 key 从 'original_linear.weight' 替换为 'weight'
            delta = lora_B @ lora_A * scale
            merged_key = key.replace('original_linear.', '')
            clean_state_dict[merged_key] = value + delta
        else:
            clean_state_dict[key] = value

    # 保存为 build_sam2 兼容格式：ckpt["model"]
    torch.save({'model': clean_state_dict}, save_path)
关键设计决策：保存时不修改内存中的模型。只在 state_dict 副本中计算合并后的权重。这样训练可以无缝继续，不会因为保存而丢失 LoRA 结构。

7. 微调参数详解

模块参数量微调策略原因
Image Encoder ~9M LoRA 微调 (r=4) 需要学会识别雷达目标，但全量微调会过拟合
LoRA 参数仅 0.11M
Mask Decoder ~4.22M 全量微调 分割头需要大幅适应雷达边界特征
参数量可控，过拟合风险低
Prompt Encoder ~0.1M 冻结 点坐标编码不需要改变
Memory Attention ~2M 冻结 帧间匹配能力是通用的
Memory Encoder ~1M 冻结 特征编码通用
